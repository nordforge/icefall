---
title: What is Icefall?
description: A fast, simple, self-hosted deployment platform
---

Icefall is a self-hosted PaaS that ships as a single Rust binary. One command to install, zero external dependencies — no PostgreSQL, no Redis, no PHP runtime.

## Who it's for

Solo developers and small teams running 1-10 apps on their own servers. If you want the simplicity of Heroku with the control of self-hosting, without burning half your VPS memory on platform overhead.

## How it's different

| | Icefall | Traditional PaaS |
|---|---|---|
| Install | Single binary + SQLite | PHP/Node + PostgreSQL + Redis |
| Memory at idle | ~30 MB | 300-500 MB |
| Static sites | Native (no Docker) | Docker container |
| Multi-server | WebSocket agents | SSH per operation |
| AI integration | 27 MCP tools | None |
| Self-update | Atomic swap + rollback | Docker rebuild |

## System requirements

| Requirement | Minimum | Recommended |
|---|---|---|
| OS | Ubuntu 22.04+ / Debian 12+ / CentOS 9+ | Ubuntu 24.04 |
| RAM | 1 GB | 2 GB+ |
| Disk | 10 GB | 20 GB+ |
| Container runtime | Docker 24+ or Podman 4+ | Docker 27+ |
| CPU | 1 core | 2+ cores |

## Architecture

**Single server**: Icefall binary + SQLite + Caddy + Docker/Podman — all on one machine.

**Multi-server**: A control plane coordinates worker servers via WebSocket agents. Each worker runs its own Docker/Podman + Caddy.

## Three interfaces

1. **Dashboard** — browser-based UI for visual management
2. **CLI** — `icefall deploy`, `icefall apps list`, `icefall logs`
3. **MCP Server** — 27 tools for AI-assisted deployment via Claude Code, Cursor, or any MCP client

## One-liner install

```bash
curl -fsSL https://icefall.dev/install.sh | bash
```

This will:
1. Detect your OS and architecture
2. Detect or install a container runtime — Docker or Podman
3. Install Caddy (for automatic HTTPS)
4. Download the Icefall binary
5. Generate a config file with a random encryption key
6. Create and start a systemd service
7. Print your dashboard URL

## Install flags

The installer accepts flags and environment variables. Flags can appear in any
order after `bash -s --`.

| Flag | Environment variable | Values | Default | Purpose |
|---|---|---|---|---|
| `--runtime=NAME` | `ICEFALL_RUNTIME` | `docker`, `podman`, `auto` | `auto` | Which container runtime to use |
| `--yes` | — | — | off | Non-interactive: accept all defaults, ask nothing |
| — | `ICEFALL_VERSION` | version tag, or `latest` | `latest` | Which Icefall release to install |
| — | `NO_COLOR` | any non-empty value | off | Disable colored output |

## Choosing a container runtime

Icefall runs your apps as containers and works with either **Docker** or
**Podman** (including rootless Podman). By default the installer auto-detects
what is available; you can also pick one explicitly.

<Tabs>
<TabItem label="Auto-detect (default)">
```bash
curl -fsSL https://icefall.dev/install.sh | bash
```

The installer looks for a running runtime, preferring one already in use. If
**both** Docker and Podman are installed it asks which to use, with an
*"auto-detect — pick for me"* fallback. If neither is installed it prompts you
to choose one to install.
</TabItem>
<TabItem label="Force Podman">
```bash
curl -fsSL https://icefall.dev/install.sh | bash -s -- --runtime=podman
```

Use this when a server has both runtimes but you want Icefall to commit to
Podman. The installer will **not** silently fall back to Docker — if Podman is
missing it installs Podman, and if Podman cannot start it stops with an error
explaining why. Requires Podman 4.0 or newer.
</TabItem>
<TabItem label="Force Docker">
```bash
curl -fsSL https://icefall.dev/install.sh | bash -s -- --runtime=docker
```

Pins the install to Docker even if Podman is also present.
</TabItem>
</Tabs>

The runtime can also be set with an environment variable, which is convenient
for cloud-init or CI:

```bash
ICEFALL_RUNTIME=podman curl -fsSL https://icefall.dev/install.sh | bash -s -- --yes
```

:::note[Re-installs keep your runtime]
If Icefall is already installed, re-running the installer keeps the runtime
recorded in your existing `config.toml`. Pass `--runtime=` explicitly only if
you intend to switch.
:::

## Requirements

- **OS:** Ubuntu 20.04+, Debian 11+, Fedora 38+, CentOS 9+, Arch
- **Architecture:** x86_64 or aarch64
- **RAM:** 1 GB minimum, 2 GB recommended
- **Disk:** 10 GB minimum
- **Ports:** 80 and 443 open for web traffic, 3000 for the dashboard

## Manual install

<Tabs>
<TabItem label="Ubuntu/Debian">
```bash
# Install Docker
curl -fsSL https://get.docker.com | sh

# Install Caddy
sudo apt install -y caddy

# Download Icefall (use the URL matching your architecture)
# x86_64:
sudo curl -fsSL -o /usr/local/bin/icefall \
  https://github.com/nordforge/icefall/releases/latest/download/icefall-x86_64-unknown-linux-gnu
# aarch64 (ARM):
# sudo curl -fsSL -o /usr/local/bin/icefall \
#   https://github.com/nordforge/icefall/releases/latest/download/icefall-aarch64-unknown-linux-gnu
sudo chmod +x /usr/local/bin/icefall

# Initialize
sudo icefall init

# Start
sudo systemctl enable icefall
sudo systemctl start icefall
```
</TabItem>
<TabItem label="Fedora/RHEL">
```bash
# Install Docker
sudo dnf install -y docker
sudo systemctl enable --now docker

# Install Caddy
sudo dnf copr enable @caddy/caddy
sudo dnf install -y caddy

# Download Icefall (use the URL matching your architecture)
# x86_64:
sudo curl -fsSL -o /usr/local/bin/icefall \
  https://github.com/nordforge/icefall/releases/latest/download/icefall-x86_64-unknown-linux-gnu
# aarch64 (ARM):
# sudo curl -fsSL -o /usr/local/bin/icefall \
#   https://github.com/nordforge/icefall/releases/latest/download/icefall-aarch64-unknown-linux-gnu
sudo chmod +x /usr/local/bin/icefall
sudo icefall init
sudo systemctl enable --now icefall
```
</TabItem>
</Tabs>

## Non-interactive install

For automated setups (CI, Terraform, cloud-init):

```bash
curl -fsSL https://icefall.dev/install.sh | bash -s -- --yes
```

Combine `--yes` with `--runtime=` to pin the runtime in an unattended install:

```bash
curl -fsSL https://icefall.dev/install.sh | bash -s -- --yes --runtime=podman
```

## Verify

```bash
icefall status
```

Should show server version, CPU, memory, and disk usage.

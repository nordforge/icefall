## Endpoints

Icefall's MCP server exposes these JSON-RPC endpoints:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/mcp/tools` | GET | List all available tools |
| `/api/v1/mcp/call` | POST | Execute a tool |
| `/api/v1/mcp/resources` | GET | List browsable resources |
| `/api/v1/mcp/resources/read` | POST | Read a resource by URI |
| `/api/v1/mcp/prompts` | GET | List prompt templates |
| `/api/v1/mcp/prompts/get` | POST | Get a prompt with arguments |

## Authentication

All endpoints require a Bearer token:

```
Authorization: Bearer icefall_your_api_token
```

Create tokens at **Settings > API Tokens** in the Icefall dashboard.

## Tool Call Example

```bash
curl -X POST https://your-instance.com/api/v1/mcp/call \
  -H "Authorization: Bearer icefall_abc123" \
  -H "Content-Type: application/json" \
  -d '{"tool": "list_apps", "params": {}}'
```

Response:
```json
{
  "result": {
    "apps": [
      { "id": "abc123", "name": "my-api", "status": "running", ... }
    ]
  }
}
```

## Resource Read Example

```bash
curl -X POST https://your-instance.com/api/v1/mcp/resources/read \
  -H "Authorization: Bearer icefall_abc123" \
  -H "Content-Type: application/json" \
  -d '{"uri": "icefall://apps"}'
```
